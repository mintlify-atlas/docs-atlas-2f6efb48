---
title: "cross — Zero Setup Cross Compilation for Rust"
description: "Cross compile and cross test Rust crates for any target architecture using Docker or Podman containers — no system configuration required."
---

`cross` gives you a drop-in replacement for `cargo` that transparently cross-compiles and cross-tests your Rust crates for any of 50+ supported target architectures, all without modifying your system toolchain.

```bash
# Build for ARM64 Linux — it just works
cross build --target aarch64-unknown-linux-gnu

# Test on MIPS64 using QEMU — this also just works
cross test --target mips64-unknown-linux-gnuabi64
```

## Why cross?

<CardGroup cols={2}>
  <Card title="Zero system setup" icon="circle-check">
    No cross-toolchain installation required. All compilers and sysroots live inside pre-built Docker images.
  </Card>
  <Card title="Identical to cargo" icon="equals">
    Use the same commands you already know. `cross build` works exactly like `cargo build`.
  </Card>
  <Card title="Cross testing" icon="flask">
    Run your test suite on any supported target via QEMU emulation — catch architecture-specific bugs before deployment.
  </Card>
  <Card title="Flexible configuration" icon="gear">
    Fine-tune behavior per-target with `Cross.toml`, environment variables, or your existing `Cargo.toml`.
  </Card>
</CardGroup>

## Quick start

<Steps>
  <Step title="Install Rust via rustup">
    ```bash
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    ```
  </Step>
  <Step title="Install cross">
    ```bash
    cargo install cross --git https://github.com/cross-rs/cross
    ```
  </Step>
  <Step title="Install Docker or Podman">
    Install [Docker](https://www.docker.com/products/docker-desktop/) or [Podman](https://podman.io/getting-started/installation) as your container engine.
  </Step>
  <Step title="Cross compile your first crate">
    ```bash
    cross build --target aarch64-unknown-linux-gnu
    ```
    `cross` automatically pulls the right Docker image and Rust target — no further setup needed.
  </Step>
</Steps>
