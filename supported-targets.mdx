## What "supported" means

A target is considered supported if `cross` can cross-compile a non-trivial binary crate (typically Cargo itself) for that target. This means the container image ships a working cross toolchain, sysroot, and cross-compiled C libraries for that triple.

## Test support

The `test` column in the table below indicates whether `cross test` works for that target. Test support relies on [QEMU](https://www.qemu.org/) user-mode emulation: the compiled foreign binary is executed by QEMU inside the container. A target has a ✓ in the `test` column if it can successfully run the [`compiler-builtins`](https://github.com/rust-lang-nursery/compiler-builtins) test suite.

<Warning>
  `cross test` runs unit tests **sequentially** because QEMU becomes unstable when multiple threads are spawned. Tests that spawn threads are more likely to fail or never terminate.
</Warning>

<Note>
  Cross testing also requires a Linux kernel with `binfmt_misc` support so the host can transparently hand off foreign-architecture binaries to QEMU.
</Note>

## Footnotes

**[1]** libc = bionic. Only works with native tests — tests that do not depend on the Android Runtime. For `i686`, some tests may fail with `assertion failed: signal(libc::SIGPIPE, libc::SIG_IGN) != libc::SIG_ERR` (see [issue #140](https://github.com/cross-rs/cross/issues/140)).

**[2]** No `std` component available.

**[3]** For some \*BSD and Solaris targets, the `libc` column indicates the OS release version from which libc was extracted.

**[4]** libc = newlib.

**[5]** Must set `image = "ghcr.io/cross-rs/x86_64-unknown-linux-gnu:main-centos"` in `Cross.toml` under `[target.x86_64-unknown-linux-gnu]` to use the CentOS 7-compatible image.

**[6]** libc = emscripten and GCC = clang.

**[7]** Must set `image = "ghcr.io/cross-rs/aarch64-unknown-linux-gnu:main-centos"` in `Cross.toml` under `[target.aarch64-unknown-linux-gnu]` to use the CentOS 7-compatible image.

## Additional targets via cross-toolchains

Pre-built images are only shipped for the targets in the table above. The [cross-toolchains](https://github.com/cross-rs/cross-toolchains) repository contains additional Dockerfiles for targets that cannot be shipped as pre-built images, including **MSVC** and **Apple Darwin** targets. You will need to build those images yourself.
