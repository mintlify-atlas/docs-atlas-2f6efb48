title: "Config File"

---

You can place a `Cross.toml` file in the root of your Cargo project, or use the `CROSS_CONFIG` environment variable to point to a config file elsewhere. All settings can also be embedded in `Cargo.toml` under `[package.metadata.cross]` (single-package crates) or `[workspace.metadata.cross]` (workspace crates).

<Note>
  The `[build]` table in `Cross.toml` is identical to `[package.metadata.cross.build]` or `[workspace.metadata.cross.build]` in `Cargo.toml`. The table structure is the same for every key.
</Note>

For annotated examples, see the upstream
[Cross.toml example](https://github.com/cross-rs/wiki_assets/blob/main/Configuration/Cross.toml) and
[Cargo.toml example](https://github.com/cross-rs/wiki_assets/blob/main/Configuration/Cargo.toml).

## `[build]`

Global build settings applied to all targets unless overridden by a `[target.TARGET]` section.

```toml
[build]
build-std = false                              # do not build the std library
zig = false                                    # do not use zig cc for the builds
default-target = "x86_64-unknown-linux-gnu"    # use this target if none is explicitly provided
pre-build = [                                  # additional commands to run prior to building the package
    "dpkg --add-architecture $CROSS_DEB_ARCH",
    "apt-get update && apt-get --assume-yes install libssl-dev:$CROSS_DEB_ARCH"
]
```

<Note>
  `$CROSS_DEB_ARCH` is automatically provided by cross and resolves to the Debian architecture name matching the compilation target.
</Note>

| Key | Type | Description |
|-----|------|-------------|
| `build-std` | `bool` | Whether to build the Rust standard library from source. |
| `zig` | `bool \| string \| table` | Enable `zig cc` as the cross-compiler. See [`[build.zig]`](#buildzig) for details. |
| `default-target` | `string` | The target triple to use when no `--target` flag is passed. |
| `pre-build` | `string \| list[string]` | Commands or a script to run inside the container before building. |

## `[build.env]`

Control which environment variables and volumes are visible inside the build container for all targets.

```toml
[build.env]
volumes = ["VOL1_ARG", "VOL2_ARG=/path/to/volume"]
passthrough = ["VAR1_ARG", "VAR2_ARG=VALUE"]
```

| Key | Type | Description |
|-----|------|-------------|
| `volumes` | `list[string]` | Host paths to mount inside the container. Entries without `=` use the env var value as the host path. Entries with `=` use the given path directly. |
| `passthrough` | `list[string]` | Host environment variables to forward into the container. Entries without `=` use the current host value. Entries with `=` set a fixed value regardless of the host. |

**Example — passthrough value semantics:**

- `"VAR1_ARG"` → forwards `$VAR1_ARG` from the host shell as-is.
- `"VAR2_ARG=VALUE"` → always sets `VAR2_ARG=VALUE` inside the container, ignoring the host value.

## `[build.dockerfile]`

Provide a custom Docker image built from a local Dockerfile, applied to all targets (unless a `[target.TARGET.dockerfile]` overrides it).

You can use a shorthand string or a full table:

```toml
# Shorthand — equivalent to setting build.dockerfile.file
[build]
dockerfile = "./Dockerfile"
```

```toml
# Full table
[build.dockerfile]
file = "./Dockerfile"         # the Dockerfile to use, relative to Cargo.toml
context = "."                 # build context directory (default: ".")
build-args = { ARG1 = "foo" } # Docker build ARGs passed at image build time
```

| Key | Type | Description |
|-----|------|-------------|
| `file` | `string` | Path to the Dockerfile, relative to `Cargo.toml`. |
| `context` | `string` | Docker build context directory. Defaults to `"."`. |
| `build-args` | `table` | Key-value pairs passed as `--build-arg` to the container build command. |

**Recommended Dockerfile base:**

Base your custom image on the default cross image for the target so you don't have to set up the cross-C toolchain yourself:

```dockerfile
FROM ghcr.io/cross-rs/aarch64-unknown-linux-gnu:latest

RUN dpkg --add-architecture $CROSS_DEB_ARCH && \
    apt-get update && \
    apt-get install --assume-yes libfoo:$CROSS_DEB_ARCH
```

Alternatively, use the `CROSS_BASE_IMAGE` build arg that cross automatically provides:

```dockerfile
ARG CROSS_BASE_IMAGE
FROM $CROSS_BASE_IMAGE
RUN ...
```

## `[build.zig]`

Use `zig cc` as the cross-compiler. This enables cross-compilation to many architectures and glibc versions from a single Docker image.

<Warning>
  `zig cc` only supports a subset of Linux GNU targets. For unsupported targets, prefer setting `zig` inside `[target.TARGET.zig]` where you have per-target control.
</Warning>

There are three equivalent shorthand forms and a full table form:

```toml
# String shorthand — sets zig.version and zig.enable = true
[build]
zig = "2.17"

# Boolean shorthand — sets zig.enable
[build]
zig = true

# Full table
[build.zig]
enable = true    # enable or disable the use of zig cc
version = "2.17" # glibc version to target
image = "myimage" # custom image containing zig (optional)
```

| Key | Type | Description |
|-----|------|-------------|
| `enable` | `bool` | Whether to use `zig cc`. |
| `version` | `string` | The glibc version to target (e.g., `"2.17"`). |
| `image` | `string` | A custom container image that provides the `zig` binary. |

## `[target.TARGET]`

Target-specific settings that override the corresponding `[build]` values. Replace `TARGET` with the Rust target triple (e.g., `aarch64-unknown-linux-gnu`).

```toml
[target.aarch64-unknown-linux-gnu]
build-std = ["core", "alloc"]   # always build core and alloc from source
image = "test-image"            # use a different image for this target
runner = "qemu-user"            # wrapper to run the binary (qemu-system, qemu-user, or native)
```

| Key | Type | Description |
|-----|------|-------------|
| `build-std` | `bool \| list[string]` | Override `build-std` for this target. Can be a bool or an explicit list of crates. |
| `image` | `string \| table` | Container image to use. See [`[target.TARGET.image]`](#targettargetimage) for advanced syntax. |
| `runner` | `string` | How to run compiled binaries: `qemu-system`, `qemu-user`, or `native`. |

## `[target.TARGET.pre-build]`

Run commands or a script inside the container before the build step for a specific target.

**Script file** (path relative to workspace root, or current directory if `--manifest-path` is used):

```toml
[target.aarch64-unknown-linux-gnu]
pre-build = "./scripts/my-script.sh"
```

The script is invoked as `RUN ./pre-build-script $CROSS_TARGET` inside the Dockerfile, where `$CROSS_TARGET` is the target triple.

```bash
#!/usr/bin/env bash
apt-get install libssl-dev -y
```

**Inline command list:**

```toml
[target.aarch64-unknown-linux-gnu]
pre-build = [
    "dpkg --add-architecture $CROSS_DEB_ARCH",
    "apt-get update",
    "apt-get install --assume-yes libfoo:$CROSS_DEB_ARCH"
]
```

<Tip>
  For more complex pre-build logic, consider using `[target.TARGET.dockerfile]` instead to directly control the Dockerfile.
</Tip>

## `[target.TARGET.image]`

Specify a custom container image for a target.

```toml
[target.aarch64-unknown-linux-gnu]
image = "my/image:latest"
```

Standard container resolution rules apply:

- If the image exists locally, it is used directly.
- Otherwise the registry (Docker Hub or specified registry) is queried.
- If only a name with no registry prefix is given (e.g., `image:latest`), Docker Hub is not searched.
- If no tag is provided, `latest` is assumed.

**Shorthand forms for the default cross image:**

```toml
[target.aarch64-unknown-linux-gnu]
# Uses ghcr.io/cross-rs/aarch64-unknown-linux-gnu:edge
image = ":edge"

[target.x86_64-unknown-linux-musl]
# Uses the image pinned to a specific digest
image = "@sha256:77db671d8356a64ae72a3e1415e63f547f26d374fbe3c4762c1cd36c7eac7b99"

[target.x86_64-unknown-linux-gnu]
# Uses ghcr.io/cross-rs/x86_64-unknown-linux-gnu:0.3.0-centos
image = "-centos"
```

**Table form — specify toolchain platforms:**

```toml
[target.aarch64-unknown-linux-gnu]
image.name = "alpine:edge"
image.toolchain = [
    "x86_64-unknown-linux-musl",
    "linux/arm64=aarch64-unknown-linux-musl"
]
```

## `[target.TARGET.env]`

Override environment variable and volume settings for a specific target. Has the same keys as [`[build.env]`](#buildenv).

```toml
[target.x86_64-unknown-linux-gnu.env]
volumes = ["VOL1_ARG", "VOL2_ARG=/path/to/volume"]
passthrough = ["VAR1_ARG", "VAR2_ARG=VALUE"]
```

| Key | Type | Description |
|-----|------|-------------|
| `volumes` | `list[string]` | Host paths to mount, scoped to this target. |
| `passthrough` | `list[string]` | Environment variables to forward, scoped to this target. |

---

## `[target.TARGET.dockerfile]`

Provide a custom Dockerfile for a specific target, overriding `[build.dockerfile]` for that target.

```toml
# Shorthand
[target.aarch64-unknown-linux-gnu]
dockerfile = "./Dockerfile"

# Full table
[target.aarch64-unknown-linux-gnu.dockerfile]
file = "./Dockerfile"         # the Dockerfile to use, relative to Cargo.toml
context = "."                 # build context directory (default: ".")
build-args = { ARG1 = "foo" } # Docker build ARGs
```

| Key | Type | Description |
|-----|------|-------------|
| `file` | `string` | Path to the Dockerfile, relative to `Cargo.toml`. |
| `context` | `string` | Docker build context directory. Defaults to `"."`. |
| `build-args` | `table` | Key-value pairs passed as `--build-arg` to the container build command. |

Enable `zig cc` for a specific target. Has the same keys as [`[build.zig]`](#buildzig).

```toml
# String shorthand — sets zig.version and zig.enable = true
[target.aarch64-unknown-linux-gnu]
zig = "2.17"

# Boolean shorthand
[target.aarch64-unknown-linux-gnu]
zig = true

# Full table
[target.aarch64-unknown-linux-gnu.zig]
enable = true     # enable or disable the use of zig cc
version = "2.17"  # glibc version to target
image = "myimage" # custom image containing zig (optional)
```

| Key | Type | Description |
|-----|------|-------------|
| `enable` | `bool` | Whether to use `zig cc` for this target. |
| `version` | `string` | The glibc version to target (e.g., `"2.17"`). |
| `image` | `string` | A custom container image that provides the `zig` binary. |
